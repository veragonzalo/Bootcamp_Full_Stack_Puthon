Metadatos, título, codificación y comentarios: la “ficha técnica” de tu página

En esta página veremos cómo los metadatos, el título, la codificación UTF-8 y los comentarios convierten tu HTML en algo más profesional, entendible para navegadores, buscadores y otros desarrolladores.

1. ¿Qué son los metadatos en HTML?

Los metadatos son datos sobre el documento, no el contenido que el usuario ve directamente en la página. Informan a navegadores, buscadores, redes sociales y herramientas de análisis sobre cosas como:

  • Codificación de caracteres (por ejemplo, UTF-8).
  • Autor y descripción del documento.
  • Configuración para móviles (viewport).
  • Recursos externos como CSS o íconos.

Todos estos metadatos viven dentro del elemento <head>, que forma parte del “esqueleto” básico de la página HTML5.

2. El elemento <title>: el nombre oficial de tu página

El elemento <title> define el título del documento, que se muestra en:

  • La pestaña del navegador.
  • El historial de navegación.
  • Los marcadores o favoritos.
  • Muchos resultados de búsqueda (como título del enlace).

Ejemplo dentro de <head>: 

<head>
  <meta charset="utf-8">
  <title>Portafolio de Ana - Desarrolladora Web</title>
</head>

Importante:

  • Debe haber solo un <title> por documento.
  • El <title> no es lo mismo que el <h1> (título visible en el cuerpo).

3. Codificación de caracteres: <meta charset="utf-8">

La codificación de caracteres define cómo se convierten letras, símbolos y emojis en números que la computadora entiende. En la web moderna, la recomendación (y práctica estándar) es UTF-8.

En HTML5 se declara así dentro de <head>:

<meta charset="utf-8">

¿Por qué es tan importante?

  • Permite mostrar correctamente tildes, eñes y caracteres especiales.
  • Evita símbolos raros como Ã± o Ã³.
  • Es la codificación más usada en la web actual.

4. Otros metadatos clave: descripción, autor y viewport

Además del charset y el título, es muy común ver otras metaetiquetas que enriquecen la “ficha técnica” de tu página.

4.1. Descripción de la página

<meta name="description"
      content="Portafolio de Ana, desarrolladora web junior enfocada en HTML, CSS y Python.">

Esta descripción puede aparecer en los resultados de búsqueda como resumen de tu página. Debe ser clara, breve y atractiva.

4.2. Autor del documento

<meta name="author" content="Ana Desarrolladora">

Indica la persona o entidad responsable del contenido del documento.

4.3. Viewport (móviles)

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Esto le indica al navegador móvil que el ancho del “lienzo” debe coincidir con el ancho del dispositivo, y define el nivel de zoom inicial. Es clave para diseños responsive.

5. Comentarios en HTML: <!-- ... -->

Los comentarios sirven para dejar notas en el código que el navegador ignora por completo. Son útiles para explicar secciones, dejar recordatorios o desactivar temporalmente algún bloque.

La sintaxis es:

<!-- Este es un comentario en HTML -->

Ejemplo práctico en una sección

<!-- Sección principal de noticias -->
<section id="noticias">
  <h2>Noticias recientes</h2>

  <!-- Artículo destacado -->
  <article>
    <h3>Comienza el bootcamp de desarrollo web</h3>
    <p>Detalles sobre el inicio del bootcamp...</p>
  </article>
</section>

Buenas prácticas con comentarios:

  • Úsalos para explicar bloques, no cada línea.
  • No pongas información sensible (contraseñas, datos privados).
  • Actualízalos o elimínalos si cambias el código, para evitar confusiones.

Desactivar temporalmente una sección con comentarios

<!--
<section>
  <h2>Sección en mantenimiento</h2>
  <p>Este contenido se mostrará más adelante.</p>
</section>
-->

El navegador ignorará todo lo que está dentro del comentario, como si no existiera en el documento.

6. index.html: la puerta de entrada por defecto

En muchos servidores web, cuando alguien visita un dominio o carpeta sin especificar un archivo (por ejemplo, https://misitio.com/), el servidor busca automáticamente un archivo de entrada, y el nombre más común es index.html.

Eso significa que si tienes un proyecto con esta estructura: 

mi-sitio/
  index.html
  contacto.html
  img/
  css/

Lo más probable es que index.html se cargue al entrar a https://misitio.com/ sin que el usuario tenga que escribirlo. Por eso, solemos considerar a index.html como la “home” o página principal de un sitio.
