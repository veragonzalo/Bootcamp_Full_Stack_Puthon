Cómo se escribe HTML: sintaxis clara y buenas prácticas

En esta página veremos ejemplos prácticos de cómo escribir HTML ordenado, bien anidado y fácil de leer, tal como se recomienda en el Tema 6 del manual.

1. Qué le importa al navegador y qué nos importa a nosotros

El navegador necesita que las etiquetas estén correctamente escritas, pero no le importa tanto si el código está en una sola línea o bien ordenado. En cambio, a nosotros sí nos importa que el HTML sea legible.

Ejemplo de código válido pero desordenado

Este bloque funciona, pero es muy incómodo de leer y mantener:

<body><header><h1>Mi sitio</h1></header><main><section><h2>Noticias</h2><p>Texto de prueba</p></section></main></body>

El navegador puede entender esto, pero para un humano es muy difícil ver la estructura. Por eso aplicamos buenas prácticas de sintaxis: sangrías, saltos de línea y orden.

2. Indentación y espacios en blanco

Una de las recomendaciones del manual es usar una sangría consistente (por ejemplo, 2 espacios) para mostrar qué elementos están dentro de otros.

Código mejorado con buena indentación

<body>
  <header>
    <h1>Mi sitio</h1>
  </header>

  <main>
    <section>
      <h2>Noticias</h2>
      <p>Texto de prueba</p>
    </section>
  </main>
</body>

3. Anidación correcta de etiquetas

La anidación correcta es fundamental. Si abres un elemento dentro de otro, debes cerrarlo antes de cerrar el elemento padre.

Ejemplo de anidación incorrecta

<p>Este es un <strong>text o importante</p></strong>

Y ahora, la versión correcta:

<p>Este es un <strong>texto importante</strong></p>

4. Buenas prácticas con atributos

Los atributos se escriben dentro de la etiqueta de apertura, normalmente siguiendo el formato nombre="valor" y en minúsculas.

Ejemplo de etiqueta con varios atributos bien escritos

<a href="https://misitio.com"
   id="enlace-inicio"
   class="btn btn-primario"
   title="Volver a la página de inicio">
  Ir al inicio
</a>

Ejemplo con atributo booleano

Algunos atributos son booleanos: basta con su presencia para indicar "verdadero". 

<input type="checkbox" checked> Acepto los términos

5. Elementos vacíos (void elements)

Algunos elementos de HTML no tienen contenido interno y, por lo tanto, no usan etiqueta de cierre. Se les llama elementos vacíos (void).

Ejemplos típicos:

  • <img> para imágenes.
  • <br> para saltos de línea.
  • <hr> para una línea horizontal.
  • <meta> y <link> en el <head>.

Ejemplo correcto de uso

<img src="img/logo.png" alt="Logotipo del sitio">
<br>
<hr>

6. Comentarios para organizar y documentar el código

Tal como se explica en el manual, los comentarios son notas para humanos que el navegador ignora. Se escriben con la sintaxis: <!-- comentario -->.

Ejemplo de comentarios útiles

<!-- Sección principal de noticias -->
<section id="noticias">
  <h2>Noticias recientes</h2>

  <!-- Artículo 1 -->
  <article>
    <h3>Lanzamiento del nuevo curso</h3>
    <p>Detalles del lanzamiento...</p>
  </article>

  <!-- Fin de sección de noticias -->
</section>

7. Resumen de buenas prácticas de estilo

  • Usar minúsculas en nombres de etiquetas y atributos.
  • Cerrar siempre las etiquetas que lo requieran, como <p>, <li>, <section>.
  • Usar indentación consistente (ej. 2 espacios por nivel).
  • No abusar de <div> cuando existen etiquetas semánticas (<header>, <main>, etc.).
  • Evitar llenar todo de style="..."; preferir CSS en archivos separados.
  • Usar nombres de clases que describan el contenido (ej. noticia-principal) y no solo la apariencia (ej. texto-rojo).
