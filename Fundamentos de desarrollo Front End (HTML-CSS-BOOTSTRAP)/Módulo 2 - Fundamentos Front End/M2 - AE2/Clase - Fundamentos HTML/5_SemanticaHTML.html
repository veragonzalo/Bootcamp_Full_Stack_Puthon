1. ¿Qué es la estructura semántica global?

La estructura semántica global es la forma de dividir tu página web en zonas con significado: cabecera, navegación, contenido principal, secciones de artículos, barra lateral y pie de página.

Para eso, HTML5 nos da etiquetas como:

  • <header> – cabecera (del sitio o de una sección).
  • <nav> – navegación principal o importante.
  • <main> – contenido principal del documento.
  • <section> – sección temática de contenido.
  • <article> – contenido independiente (artículo, post, noticia).
  • <aside> – contenido complementario o lateral.
  • <footer> – pie de página (del sitio o de una sección).

6. ¿Cuándo usar <div> y cuándo etiquetas semánticas?

La regla general recomendada en el manual es: usa etiquetas semánticas siempre que exista una que describa bien el contenido.

Por ejemplo:

  • Usa <header> en vez de <div id="header">.
  • Usa <nav> en vez de <div class="menu"> para menús principales.
  • Usa <main> para el contenido principal.
  • Usa <article> para posts, noticias o fichas de contenido independiente.

<div> sigue siendo útil como contenedor genérico cuando:

  • No existe una etiqueta semántica adecuada.
  • Necesitas un wrapper extra solo para aplicar estilos o layout con CSS.
