Estructura básica de un documento HTML5: el esqueleto de tu página

En esta página vamos a ver, paso a paso, cómo se arma el “esqueleto” de cualquier documento HTML5 moderno: <!DOCTYPE html>, <html>, <head> y <body>.

1. Vista general del esqueleto HTML5

Todo documento HTML5 bien formado sigue una estructura básica: 

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="utf-8">
    <title>Título de la página</title>
  </head>
  <body>
    <h1>Hola mundo</h1>
    <p>Esta es mi primera página HTML5 completa.</p>
  </body>
</html>

2. La declaración <!DOCTYPE html>

La primera línea de un documento HTML5 moderno es la declaración: 

<!DOCTYPE html>

Su función principal es indicarle al navegador que el documento está escrito siguiendo el estándar HTML5 y que debe renderizarlo en modo estándar, evitando comportamientos antiguos o de compatibilidad (“quirks mode”).

3. El elemento raíz <html> y el atributo lang

Después del DOCTYPE viene el elemento raíz: 

<html lang="es">
  ...
</html>

Aquí ocurren dos cosas importantes:

  • <html> ... </html> envuelve todo el contenido del documento HTML.
  • El atributo lang="es" indica que el idioma principal del documento es el español. Esto ayuda a lectores de pantalla, buscadores y herramientas de accesibilidad.

4. La sección <head>: metadatos y configuración

El elemento <head> contiene metadatos, es decir, datos sobre el documento que no se muestran directamente como contenido en la página.

Ejemplo de <head> mínimo recomendado

<head>
  <meta charset="utf-8">
  <title>Mi primera página HTML5</title>
</head>
  • <meta charset="utf-8"> establece la codificación de caracteres. UTF-8 es el estándar moderno recomendado.
  • <title>...</title> define el título del documento que verás en la pestaña del navegador, en los marcadores y en muchos resultados de búsqueda.

5. La sección <body>: el contenido visible

Todo el contenido que el usuario ve en la página (texto, imágenes, enlaces, listas, formularios, etc.) se coloca dentro del elemento <body>.

Ejemplo sencillo de <body>

<body>
  <h1>Bienvenido a mi página</h1>
  <p>Este es el contenido visible para el usuario.</p>
</body>

Dentro del <body> también utilizamos etiquetas semánticas como <header>, <main>, <section> y <footer> para organizar el contenido, como se vio en otros temas.

6. Elementos vacíos dentro del esqueleto

Algunos elementos HTML no tienen contenido interno y por eso no usan etiqueta de cierre. Son los llamados elementos vacíos.

Ejemplos en <head>

<meta charset="utf-8">
<link rel="stylesheet" href="styles.css">

Ejemplos en <body>

<img src="img/logo.png" alt="Logotipo del sitio">
<br>
<hr>

7. Construyendo tu index.html paso a paso

Imagina que estás en WebStorm y creas un nuevo archivo llamado index.html. Sigamos la construcción del documento:

Paso 1: agregar el DOCTYPE

<!DOCTYPE html>

Paso 2: agregar el elemento <html> con el idioma

<!DOCTYPE html>
<html lang="es">

</html>

Paso 3: agregar <head> y <body>

<!DOCTYPE html>
<html lang="es">
  <head>

  </head>
  <body>

  </body>
</html>

Paso 4: completar el <head> mínimo

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="utf-8">
    <title>Mi primera página HTML5</title>
  </head>
  <body>

  </body>
</html>

Paso 5: agregar contenido básico al <body>

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="utf-8">
    <title>Mi primera página HTML5</title>
  </head>
  <body>
    <h1>Hola mundo</h1>
    <p>Esta es mi primera página HTML5 completa.</p>
  </body>
</html>
